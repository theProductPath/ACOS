# Voice analysis — heuristics

Goal: produce a concise, useful summary of how the client *sounds* so any agent later producing copy in their voice has enough to work from. The summary is not a substitute for the full guideline generation a dedicated voice plugin can produce; it is the floor.

## Companion-plugin check (do this first)

If the `brand-voice` plugin is detected in the running environment:

1. Pull 8–15 representative sentences from the homepage and confirmed secondary pages.
2. Hand the samples to `brand-voice:guideline-generation`.
3. Use that output as the body of `Brand/voice.md`. Add a `Generated by brand-voice on YYYY-MM-DD` line at the top.
4. Note the handoff in `Brand/_sources.md`.
5. Skip the rest of this reference.

If `brand-voice` is **not** detected, the skill produces its own lightweight summary. Continue below.

## Sample selection

Pick 8–15 sentences that span:

- **Hero copy** — the headline + subhead on the homepage (highest-signal voice).
- **Value propositions** — section headers and one-liners on the homepage.
- **Body explanation** — at least 2 paragraphs of substantive prose from About or Services.
- **CTAs** — what the buttons say ("Get started" vs "Talk to us" vs "Request a demo" all carry different voices).
- **Microcopy** — form labels, link text, footer language if distinctive.

Avoid:

- Boilerplate (privacy policy, cookie notice, ToS) — written by lawyers, not the brand.
- Third-party embeds (chat widget greetings, social card previews).
- Customer testimonials — these reflect the customer's voice, not the brand's.

## What to summarize

Write `voice.md` covering these dimensions:

| Dimension | What to capture |
|---|---|
| **Tone descriptors** | 3–5 adjectives. Choose from: formal / casual, technical / plain, warm / precise, direct / aspirational, confident / measured, playful / serious. Pick adjectives that contrast — "warm but precise" is more useful than "professional." |
| **Person and voice** | First person plural ("we"), second person ("you"), third person, or no person? Active or passive default? |
| **Vocabulary register** | Industry jargon-heavy or plain-English? Acronym density? Buzzword load? |
| **Sentence shape** | Short and punchy, long and explanatory, or mixed? Frequent fragments or full clauses? |
| **Signature phrases** | Words or phrases that recur and feel intentional — these are voice fingerprints. |
| **What the copy avoids** | Negations are valuable. "Does not use exclamation marks." "Avoids superlatives." "Never says 'cutting-edge'." |
| **Representative quotes** | 3–5 short verbatim quotes from the source, with page attribution, so a future agent can pattern-match. |

## Confidence and TODO sentinels

Voice analysis from a static skim is inherently lower confidence than color extraction. Mark **every** voice dimension with:

- **High confidence** — only when the pattern is unmistakable across all sampled pages (e.g., "every page uses 'we' and short sentences").
- **Medium confidence** — recurring but not universal.
- **Low confidence** — observed but with notable counter-examples. Add a `TODO: Verify with <principal>` sentinel.

When in doubt, prefer to **show** with quotes and let the principal name the pattern.

## What to do with the voice summary

The output is used by downstream skills (proposal drafting, email writing, deck copy) as the canonical reference for the client's voice. The principal verifies and edits before that downstream use is considered safe.

## When voice samples are thin

Some client sites are very lean — a single landing page, a few hundred words total. When this is the case:

- Note the corpus size explicitly in `voice.md`.
- Mark all dimensions at medium-or-low confidence.
- Recommend in the TODO that the principal supply additional source material (decks, emails, prior proposals) for a deeper second pass.

Do not invent voice characteristics to fill out the template. An honest "we couldn't tell" is more useful than a confident guess.
